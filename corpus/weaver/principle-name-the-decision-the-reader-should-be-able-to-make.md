title: Name the decision the reader should be able to make
type: principle
---

Before sketching anything, write one sentence: *after reading this,
[audience] should be able to [decide/do X]*. If you can't finish the
sentence, the project is unfocused and the viz will wander.

Receipts: the Maplewood and New Bern coverage profiles work because
the decision is explicit — *a newsroom editor deciding what to cover
here*. `info-market-voting` works because the decision is *which lever
a civic funder should pull in this community*. Projects that drift are
ones where this sentence stayed vague.

Pattern: put the decision sentence at the top of the project's
`notes.md` (or README) before any `ingest.js` or `index.html`. It
becomes a cheap test for every later choice — *does this chart help
with that decision, or is it just interesting?*
